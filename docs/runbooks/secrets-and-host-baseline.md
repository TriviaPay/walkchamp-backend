# Secrets And Host Baseline

## Secrets Model

- OCI Vault is the source of truth.
- Runtime env files are rendered to:
  - `/etc/walkchamp/api.env`
  - `/etc/walkchamp/worker.env`
  - `/etc/walkchamp/migrate.env`
- Owner: `walkchamp:walkchamp`
- Permissions: `0600`
- Deploy scripts must never print secret values.
- Env files are not copied into release directories.
- Env files are not backed up with releases.
- Secret rotation requires a controlled service restart.
- Emergency revocation means:
  1. rotate in OCI Vault
  2. re-render env files
  3. restart affected service
  4. invalidate provider-side credentials if applicable

## Host Baseline

- Runtime user: `walkchamp`
- Deploy user: `deploy`
- SSH: key-only
- Password login: disabled
- Time sync: enabled
- Patch cadence: weekly or faster for security fixes
- Restart policy: `systemd` with automatic restart
- Core dumps: disabled unless an incident requires temporary enablement
- Firewall: deny public API port, allow LB private ingress only

## Object Storage Policy

- Bucket stays private.
- Keys are generated by the backend only.
- Prefixes are separated by asset type.
- Avatars are deleted on replacement.
- Group images are deleted on replacement.
- Orphan cleanup must remove objects no longer referenced by DB rows.
- Abandoned upload lifecycle policy must be enabled in OCI.
- Account-deletion behavior must remove owned user objects.
- Malware scanning is deferred in v1 and must be revisited before broader public launch.

## Kill Switches

These can start env-backed and later move to DB-backed feature flags:

- disable payments
- disable uploads
- disable signups
- disable worker processing
- disable push notifications
- global maintenance mode
